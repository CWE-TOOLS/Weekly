6.1

Standard Operating Procedure Inputting Staging

Rev: 1.0
Cadence: Per casting — before the work is scheduled
Owner: ____________
The mechanics of entering staging descriptions in the Project Portal — the department-by-day grid where task descriptions are written ahead of time. What you write here automatically appears on the matching tasks on the weekly schedule board. What to write for each department is covered by its own SOP — ex: Mill and Form Out Staging
1

Open the project in the Project Portal and go to the Staging tab. Staging needs castings first — the project must be saved with at least one casting in Casting Inventory.

2

Click the casting's pill at the top to bring that casting into view. On phased projects, only castings in the phase you're viewing appear.

3

For each department row, use + Add Day / Remove Day until the day columns match how long that department's work runs.

4

Drop any departments that don't apply to this casting with Remove Department; add missing ones with the + Department… drop-down.

5

Click a day to expand it, then type the description of what that department does on that day. Each line you type prints as its own line on the weekly board card.

6

Repeat down the grid — every department, every day — until the casting is fully described.

7

Repeat for each casting. For castings that repeat, fully stage one, then use Copy Staging / Paste Staging to drop the same scaffold onto the others. Pasting overwrites the target casting — the portal asks you to confirm.

8

Confirm it saved. The portal autosaves as you type — watch the save indicator near the top-right of the project header. Descriptions appear automatically on the matching weekly-board tasks, matched by casting #, department, and day order.

Bottom line: every casting is fully described in the Staging grid before its tasks land on the weekly board. SOP 6.1 — Inputting Staging
