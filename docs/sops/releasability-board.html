The Releasability Board answers one question: is this project ready to release to production? Every casting is a row; every column is a milestone that must be done before the job can move. Scan it to see at a glance what is ready (green), what is in progress (yellow), and what still needs work (red). This sheet explains everything you see and can do on the board.

1 What the board is for

Each row is one casting (a specific production job). Moving left to right, the columns are the approvals and prep steps owned by Classroom, the Project Manager, and Engineering. When every column is green — ending with the Green Sticker — the project is cleared to build and its project number turns green. The board updates live and is shared, so what you mark is seen by everyone.

2 The toolbar (top of the page)

Search projects
Type a project name or number to filter the board to matching rows. Click the × to clear.
STATUS legend
Reminder of what the four cell colors mean (see section 4).
TASKS legend
Shows which columns are owned by PM (teal) vs. Engineering (green).
Hide Completed Projects
Check to hide any project that is all-green, so you only see work that is still open.
+ Add Week
Creates a custom (manual) week you can name — for rush jobs or groupings that aren't on the calendar schedule.
Printer icon
Opens the print menu (see section 7): choose a format and which weeks to print.
"Updated just now"
How fresh the data is. Green dot = current, yellow = aging, red = stale/possible network issue.
Refresh icon
Forces an immediate sync. Use it if the board feels out of date.
Fullscreen icon
Expands the board to fill the screen — good for a wall display or large monitor.

3 Reading a row

Projects are grouped under week headers (e.g. "Week 2: Jul 6–11"). Click the ▼ / ▶ arrow on a week header to collapse or expand that week; the board remembers it on refresh. Each row shows, left to right:

0383 0814 (green = released) Cast #19 orange edge = manual project
Project number
The job's ID (e.g. 0383). Grey by default; turns green once the Green Sticker is marked complete.
Project name
Pulled from the portal by project number. If the number isn't found, the schedule name is used instead.
Cast badge
Blue tag (e.g. Cast #19, #MU1, #MOCKUP) identifying which casting/run this row is.
Start Date
Scheduled start (e.g. "Wed 24"). Manual projects with no date show "--".

To change a status, click the cell. It cycles: Incomplete → In Progress → Complete → Not Applicable → back to Incomplete. The change saves automatically and syncs to everyone. Hover a cell to see a tooltip with the project and milestone name.

Hover a project name to reveal copy and paste buttons — copy one project's whole row of statuses and paste the same pattern onto another. (Paste is greyed out until you've copied.)

6 Weeks & manual projects

Add a project
On a week header, click + Add Project. Enter a Project # (name is looked up automatically) and a Cast #.
Manual projects
Projects you add by hand show a left orange edge. Hover to reveal a drag handle and a × to delete.
Move a project
Drag a manual project by its handle onto another week header (highlights blue) to reschedule it.
Manual weeks
Custom weeks from + Add Week can be renamed, moved up/down, or deleted from the week header.

7 Printing

Click the printer icon to open the print menu, then pick a format and the weeks to include:

Releasability Report
Simple portrait table of projects and start dates, grouped by week.
Folder Board
Large landscape print for the shop board — bold names, cast numbers, dates, with released / not-released flags.
Week selection
Check individual weeks, or use the presets: Next Month, Current Month, All, None.
Print
Click Print Selected / Print Folder Board; a new tab opens the layout — use your browser's print dialog (Ctrl+P).

8 Where the data comes from

Projects and schedule info come from the Google Sheets schedule; the status cells and any manual projects are saved in the database and persist between sessions. The board auto-refreshes in the background, so other people's changes appear on their own. Switching back to the tab syncs immediately, and the Refresh icon forces a sync any time. Watch the "Updated just now" dot to confirm the board is current.

Bottom line: Rows are castings, columns are milestones. Click a cell to update its status; all green + Green Sticker = released (project number turns green). The board is live and shared. Section 1 — Releasability Board
