Statuses & Workflow

Every project carries a status that tracks where it is in its life, plus an assigned PM. Both are set on the Project Info status bar, and the status decides how and where the project is grouped.

Where status shows up

A project's status appears in three places: the Project Info status bar (where you set it), the project list, and the Releasability Board. Changing it in one place changes it everywhere.

The six statuses

The statuses below, in order, form a project's life cycle from start to finish. The team sets each one by judgment, so treat these as what each stage generally means rather than a strict rule:

Kicked Off
The job has just started. Information is being gathered and the kickoff classroom is happening.
Releasability
The job is being reviewed for readiness to release to production. This ties to the Releasability Board.
Approved
The job has been cleared to build.
In Production
Panels are actively being made and tracked.
Shipped
The work has left the shop.
Closed Out
The job is complete and archived.

How status is set

Status is set manually — nobody and nothing changes it automatically. Someone on the team picks the right stage from the dropdown on Project Info when the job moves forward. That choice controls how and where the project is grouped, both in the project list and on the Releasability Board.

How to change a project's status

  1. Open the project. Find it in the project list and click its row.
  2. Go to Project Info. It opens here by default; the status bar is across the top.
  3. Pick from the Status dropdown. Choose the new stage. It saves automatically — there's no separate save button.

The PM field

Right next to Status on the Project Info status bar is the PM field — the project manager assigned to the job. Choose a name from the dropdown (for example, Tim or Islam). The PM shows in the project list so people can sort or scan to find their own jobs.

Big jobs and phases

Large projects can be split into per-phase views on several tabs — see Phases. Status stays project-wide either way: a project has one status, no matter how many phases it has.
