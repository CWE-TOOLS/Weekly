Batch Tickets

Generate concrete batching tickets for each casting. The tab takes the recipe from the Color Log and scales it to the volume a casting needs, then prints one ticket per batch for the casting floor. There are three batch types: FACE (the face mix), FIRST BACK UP, and FINAL BACK UP.

Where it lives

It's the Batch Tickets tab in a project's left sidebar, just after the Color Log tab it depends on.

What this tab is for

A batch ticket tells the floor exactly what to weigh out and mix for one batch of concrete. Because every casting needs a different volume, the portal scales the Color Log recipe to fit and splits it into the right number of batches, then prints a clear, color-coded page for each one. You set a few inputs per casting; the generator does the math.

Before a ticket will generate

Four things must be in place: the project must be saved; a Color Log must exist (tickets scale ingredients from it); the casting needs a Total Cu Ft; and the color log needs a Sand entry. Miss any of these and the ticket won't generate.

Target Slump (project-wide)

At the project level you set a Target Slump — the inches of spread — for each batch type. These values drive the slump bullseye drawn on every printed ticket.

Batch typeDefault Target Slump
Face Mix6"
First Back Up5"
FINAL Back Up5"

Per-casting settings

Pick a casting from the casting pills, then fill in its inputs:

Total Cu Ft
Required. The total cubic feet of concrete the casting needs. The whole recipe is scaled from this. Use Import from Tracking to pull the totals in automatically — see Tracking.
Face Sq Ft
The face square footage for the casting. Can also be imported from Tracking.
Pigment reduction %
How much to cut pigment in the back-up mixes (default 50%).
Reduce pigment on First / Final Back Up
Choose whether the FIRST BACK UP and FINAL BACK UP batches get the pigment reduction applied.
Batched by
Who batched the mix. Prints on the ticket.
Notes
A notes box that prints on the ticket.

How to generate batch tickets

  1. Make sure the recipe is ready. Confirm a Color Log exists for the project and includes a Sand entry, and that the project is saved.
  2. Pick the casting. Click the casting from the casting pills.
  3. Enter the volume. Type the Total Cu Ft (required) and the Face Sq Ft, or use Import from Tracking to pull them in.
  4. Set pigment reduction. Adjust the pigment reduction % and choose whether First Back Up and Final Back Up are reduced.
  5. Review the scaled batches. Check the batches the generator produced — count, batch type, and scaled ingredients.
  6. Print. Use Print Batch Tickets to print one page per batch.

What the generator does

You don't need to do the math — the generator handles it. In plain English, for each casting it:

Splits into batches
Decides how many batches to make based on standard sand sizes.
Labels by cast method
Labels each batch by the Color Log's cast method (Spray Up vs Direct Cast).
Scales the recipe
Scales the whole recipe by sand weight to match the volume needed.
Applies material rules
Applies automatic adjustments — sand swaps, fiber-to-Cemfill conversion, and pigment reduction.
Prints color-coded pages
Prints one page per batch, color-coded by type: FACE orange, FIRST BACK UP blue, FINAL BACK UP yellow.
The full batching logic

Every batching rule and decision the floor needs is on one handout: Batching Ticket — Floor Cheat Sheet (PDF). It's the single-page reference for casting staff covering all the sand-size, material, and reduction rules the generator follows.

Printing batch tickets

The Print Batch Tickets button opens a dialog where you pick which castings to print, then prints a ticket for each batch. See Printing & Labels.

Save before you print

Save the project first so the tickets reflect your latest inputs. The portal saves automatically a moment after you type — wait for the save indicator to read Saved.

Phases

If the project uses Phases, batch tickets are handled per phase — you pick castings and generate tickets within the phase you're working in.
