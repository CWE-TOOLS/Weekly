Casting Inventory

The master parts list for the whole job. Here you list every casting on the project and the panels that make up each one. Almost everything else in the portal — Tracking, Casting Layout, Shipping, Optimizer Hours, and Batch Tickets — is built from what you enter here, so it pays to get it right.

Two words to know first

A casting is one concrete unit, identified by a casting number that is unique within the project. A casting is made of one or more panels — the individual forms (pieces) that get poured. Casting Inventory is where you enter both.

Save the project first

Make sure the project has been saved (it has a Project # on Project Info) before you start adding castings. Castings attach to the saved project, so adding them to an unsaved project can lose your work.

What this tab is for

Think of Casting Inventory as the bill of materials for the job. You build up the list of castings, and inside each casting you list its panels with their sizes, color, sealer, and quantity. From there the portal does the heavy lifting: it generates the production Tracking sheet automatically, hands panel sizes to Casting Layout, feeds Shipping, Optimizer Hours, and Batch Tickets.

The layout of the tab

Notes
A project-level box at the top for any notes about this project's castings — assumptions, customer requests, reminders for whoever picks the job up next.
Casting cards
Each casting is its own card, listed down the page. A card shows the casting number, an optional casting date, an optional description, and an inventory grid of the casting's panels.
Add buttons
+ Add Casting and + Add Remake Casting at the top create new cards (see below).

The casting card

Every casting card has a few controls:

Casting #
The casting's number, unique within this project. This is how the casting is referred to everywhere else in the portal.
Casting date
Optional — the date this casting is poured or planned for.
Description
Optional — a short label so people know what the casting is (e.g. “North wall panels”).
Inventory grid
The list of panels in this casting. Expand or collapse it with the chevron. Each row is one panel type.
Edit / Delete
Change the casting's details, or remove the casting entirely. Deleting a casting removes its panels from Tracking too.
Drag to reorder

Grab a casting card and drag it up or down to change the order castings appear in. The order you set here carries through to the other tabs.

How to add a casting

  1. Make sure the project is saved. Confirm it has a Project # on Project Info first.
  2. Click + Add Casting. A new casting card appears. The portal suggests the next casting number automatically — change it if you need to.
  3. Fill in the details. Set the casting date and a description if you want them.
  4. Add the panels. Open the inventory grid and add one row per panel type, entering the dimensions, color/material, sealer, and quantity (see the field reference below). The cubic footage is worked out for you.
  5. Done. Your changes save automatically, and Tracking updates itself to match.

Panel fields, in plain English

Each row in a casting's inventory grid records one panel type:

FieldWhat it means
Width × Length × DepthThe panel's dimensions. These sizes also drive how the panel is drawn in Casting Layout.
Color / MaterialThe color or material type for the panel — which mix it's poured in.
SealerThe sealer to be applied to the panel.
QuantityHow many of this panel type are in the casting.
Cubic footageCalculated automatically from the dimensions and quantity.
RejectedA count of how many of this panel were marked rejected. This comes from the Tracking tab — you don't set it here. It's the starting point for remakes (below).

Remakes — rebuilding rejected panels

When panels fail inspection, they're marked rejected over on Tracking. Those rejected counts show up here on the panel rows. A remake is how you cast replacements for them, and there are two ways to do it.

+ Add Remake Casting

Click + Add Remake Casting to open a dialog that builds a brand-new casting from rejects. You pick which existing castings to pull rejected panels from, and the portal creates a new casting with one inventory line per panel type, summing up the rejected quantities. The new panels get fresh IDs of their own; the original rejected panels stay marked rejected on the castings they came from. Use this when you want the remakes grouped into their own dedicated casting.

Import Remakes

Inside an existing casting there's also an Import Remakes action. It lets you bring selected rejected panels from across the project into that casting, rather than creating a separate one. Use this when the remakes belong with a casting you're already building.

Originals stay rejected

Either way, the rejected panels you pulled from stay rejected on their source casting — the remake creates new replacement panels, it doesn't erase the rejection history.

If the project uses Phases

When a project is split into phases, Casting Inventory only shows the castings that belong to the active phase. Switch the active phase to see and edit a different set of castings. Most projects aren't phased, so you'll usually see every casting at once.

This tab feeds the rest of the portal

Tracking is built automatically from what you enter here — add, remove, or change a panel and the Tracking sheet updates itself. The same panel data also drives Casting Layout, Shipping, Optimizer Hours, and Batch Tickets. Keeping the inventory accurate keeps everything downstream accurate.
