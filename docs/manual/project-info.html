Project Info

The cover sheet for the whole project. This is the first tab you land on, and it holds the identifying details every other tab and printout pulls from — project number, name, customer, dates, scope, and the project's status.

Where it lives

It's the first tab in a project's left sidebar — the Project Info tab (the ℹ️ icon). It opens automatically whenever you open or create a project.

What this tab is for

Think of Project Info as the title page of the job folder. Most of what you type here is reference information — it doesn't drive a calculation, but it shows up on printouts, labels, and at the top of every other tab so everyone knows which job they're looking at. A few fields are special: the Project # identifies the project everywhere, the Status drives where the project shows up, and the dates feed scheduling.

The status bar (top of the sheet)

Across the top of the sheet you'll find three controls plus the company mark:

Status
Where the project is in its life cycle: Kicked Off → Releasability → Approved → In Production → Shipped → Closed Out. This is the same status shown in the project list and on the Releasability Board. See Statuses & Workflow.
PM
The project manager assigned to the job. Pick from the list (e.g. Tim, Islam). Shows in the project list so you can find your own jobs quickly.
CWE mark
The Concreteworks East logo block — just branding on the sheet.

Filling out the sheet

  1. Identify the project. Enter the Project Name, Project # (required — this is the unique number used everywhere), Kickoff Date, Project Address, Estimator / Sales Manager, and Architect.
  2. Add the contacts. Fill in Main Contact Information (name, phone, company, email) and Delivery Information (site contact, phone, delivery address, site restrictions). These print on shipping paperwork.
  3. Set the dates. Need By Date accepts free text like “August 2025” — or click the calendar button to pick an exact date. Production Start Date is also free text (e.g. “July”).
  4. Describe the work. Use Scope of Work for what's being built and Imperative Information for the must-know items the whole shop needs to see.
Saving

The portal saves your typing automatically as you go — watch the save indicator near the top-right of the project header. There's no separate “Save” button to remember.

Header buttons

At the top of the project (above the tabs) you'll see action buttons that apply to the whole project:

Print
Prints the project info sheet.
Print Label
Opens the folder-label printer so you can print a label for the job folder (project name + casting # on top, color log below). See Printing & Labels.
Delete
Permanently deletes the entire project and everything in it. You'll be asked to type the project number to confirm — there's no undo.

Phases

At the bottom of Project Info is the Phases section. Most projects don't need it, but for big jobs you can split the project into phases (building wings, delivery waves, etc.) so that Castings, Tracking, Shipping, Optimizer Hours, and Batch Tickets each get their own per-phase view. Color Log and the cover sheet stay project-wide.

One-way switch

Turning on phases is permanent for that project — you can't turn it back off. Only enable it when you're sure the job is genuinely phased. Full details on the Phases page.
