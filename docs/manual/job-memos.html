Job Memos

A dated, chronological log of everything that happens on this job — decisions made, problems found, calls with the customer, changes to the scope. Think of it as the running diary of the project: a single place anyone can read to catch up on what's gone on.

Where it lives

Job Memos is the second tab in a project's left sidebar. It belongs to the one project you have open. To read memos from every project at once, use the Recent Memos Feed folder tab at the top of the portal instead.

What this tab is for

Use Job Memos to write down anything worth remembering about the job. When a decision gets made, when something changes, when a question gets answered — jot a quick memo. Months later, when someone needs to know why the job was done a certain way, the answer is right here instead of lost in someone's memory or inbox.

Save the project first

You can only add memos to a project that has already been saved. If you've just started a brand-new project, fill in the Project Info (at minimum the Project #) so the project gets saved, then come back to add memos.

The toolbar

At the top of the tab you'll find two things:

+ Add Memo
Creates a new, empty memo at the top of the list, dated today, ready for you to type into.
Memo count
A live count of how many memos this project has, so you can see at a glance how much history a job has built up.

How the list is organized

Memos are grouped under friendly date headers so the log reads like a timeline. Recent days show as Today and Yesterday; older entries get a plain date header like Wed, May 6. Each memo is a card filed under the date you gave it, with the newest activity at the top.

Adding a memo

  1. Click + Add Memo. A fresh memo card appears at the top, already dated today.
  2. Type what happened. Write your note in the body box. It grows taller as you type, so long entries are fine — there's no length limit to worry about.
  3. Check the date. If the memo is about something that happened on a different day, click the date picker and choose the right date. The memo re-files itself under that day's header.
  4. Add your name. Put who's writing in the author field so the team knows who to follow up with.
No Save button

Memos save themselves a moment after you stop typing — there's nothing to click. Watch the save indicator near the top-right of the project header to confirm your note was kept.

What's on a memo card

FieldWhat it's for
DateThe day the memo is about. Click the date picker to change it. The memo files itself under the matching date header.
BodyThe note itself — what happened, what was decided, what to remember. The box grows as you type.
AuthorWho wrote the memo, so the team knows who has the context.
DeleteRemoves that memo. Use it to clear out an entry made by mistake.
Write down the “why,” not just the “what”

The most useful memos capture decisions and the reasons behind them — “customer approved switching to the lighter mix to hit the ship date” tells the next person far more than “mix changed.” If someone new picks up this job, the memo log should let them get up to speed without having to track anyone down.

Seeing memos across all jobs

Every memo you write here also shows up in the portal-wide Recent Memos Feed — the folder tab at the top of the portal that combines memos from every project into one searchable feed. It's a quick way to scan what's happening across all active jobs. You still create and edit memos here, inside the project; the feed is just a window onto them.
